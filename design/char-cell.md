## 1. 문제 (증상 → 원인)

현재 본문 렌더링에는 '글자'라는 단위 객체가 없다. 순수 문자열 위에 음·드릴
밑줄·문법 표시를 각각 다른 방식으로 덧칠하며, 장식 하나가 켜질 때마다 다른
요소가 간섭받는다.

| # | 증상 | 원인 (위치) |
|---|------|------------|
| 1 | 전체 길이가 정확히 일치할 때만 글자별 음이 붙고, 부호·현토 하나만 껴도 카드 아래 한 줄로 떨어짐 | `cpLen(front)===cpLen(reading)` 전체 관문 — `render.ts:88·530·594` |
| 2 | 뒤집을 때(음 표시) 한자 배치가 미묘하게 재배열 | 뒤집는 순간에만 글자를 `<ruby><rt>` 구조로 재조립 + 드릴 구간에 `inline-block` 래퍼 → 글자 폭·줄바꿈 재계산 — `render.ts:262-285` |
| 3 | 드릴 밑줄 구간 안팎의 음 위치가 어긋남 | 드릴 구간만 `inline-block` + `border-bottom 2px` 래퍼 — 베이스라인 상이 |

| + | 문법 편집 모드는 음을 ruby가 아닌 별도 셀 구조로 그려 화면마다 모양이 다름 | `render.ts:118-125` |

## 2. 핵심 원칙

본문 렌더링의 기본 단위를 문자열이 아닌 **글자 셀**로 삼는다.
음(독음)·드릴 밑줄·문법 표시(SVO·구절)는 셀에 붙는 **독립 속성**이며,
어느 속성을 켜고 꺼도 다른 속성과 한자 배치에 간섭하지 않는다.

한 문장으로: **"덧칠 4겹"을 "속성을 가진 글자 셀 하나"로 바꾼다.**

## 3. 요구사항

### 배치 불변

- **R1** — 음이 나타나거나 사라져도(앞↔뒤) 한자의 가로 위치·줄바꿈은 완전
  동일. 세로(줄 간격)만 일정하게 확장.
- **R2** — 줄바꿈은 글자 단위, 앞뒷면 동일 규칙. 드릴 구간이 줄 끝에 걸리면
  글자 단위로 쪼개지고 밑줄은 다음 줄에서 이어진다.
  (현행 앞면=글자 단위 / 뒷면=통째 불일치 해소)
- **R3** — 드릴 밑줄은 한자에 밀착 고정, 음은 그 아래. 음 토글이 밑줄 위치를
  움직이지 않는다. 드릴 구간 안팎의 음 높이·위치는 동일.

### 음 정렬

- **R4** — 음은 **한자 글자에만 1:1**로 붙는다. 공백·문장부호·현토는 음 없이
  통과. 전체 길이 일치 여부와 무관하게 모든 카드에서 글자별 음이 정렬된다.
  (현행 `cpLen` 전체 관문 폐지)
- **R5** — 한자 수와 대응 음절 수가 실제로 다른 **데이터 오류 카드**는
  현행처럼 카드 아래 한 줄로 안전하게 표시하고, 빌드 lint가 해당 카드를
  경고해 데이터 수정을 유도한다. 현재 후보 12장(1절 표) — R4 규칙 적용 후
  진짜 오류만 남는다. 장기 목표 0장.
- **R6** — char 카드(reading이 "봄 춘" 훈+음 구조)는 글자별 정렬 대상이 아님.
  표시 방식은 현행(뒤집으면 아래 훈·음) 유지하되, 뒤집을 때 대형 한자가
  움직이지 않는 배치 불변만 보장.

### 드래그 선택

- **R7** — 음이 보이는 상태에서 드래그해도 선택 문자열에 음은 절대 포함되지
  않는다. 드래그한 본문 글자(한자+현토·부호)는 그대로 잡히고, 세부 수정은
  현행대로 카드 추가 모달에서.

### 적용 범위

- **R8** — 문법 보기 모드 포함: SVO 배경·레이블, 구절 표시, 음, 드릴 밑줄이
  동시에 켜져도 위 원칙이 모두 성립.
- **R9** — 문법 편집 그리드도 같은 글자 셀 표현으로 통일 (편집 기능은 유지).

## 4. 검증 기준 (완료 판정)

1. 긴 sentence 카드를 반복해 뒤집어도 한자 줄바꿈·가로 위치가 픽셀 단위 불변
2. 드릴 구간 안/밖 음 높이 동일, 음 토글 시 밑줄 부동
3. 따옴표·현토 낀 카드에서 한자별 음 정렬 확인
4. 음 켠 채 드래그 → 카드 추가 모달에 음 미포함
5. 문법 보기 + 음 + 밑줄 동시 상태 정상
6. char 카드 뒤집기 시 대형 한자 부동
7. lint 경고 카드를 데이터 수정으로 0장 달성

## 5. 재량으로 남긴 것

드릴 클릭·호버 영역에 음까지 포함할지, 음 켜짐 시 줄 간격 수치 같은 미세
디테일은 구현·목업 단계 재량.
