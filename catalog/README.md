# catalog/ — 문헌 카탈로그 (다운로드 배포용)

앱에 **베이킹되지 않고** 사용자가 홈의 "문헌 받기"에서 내려받는 문헌들.
`src/data/`와 동일한 DocJSON 스키마의 `.json` 파일을 이 폴더에 넣고 push하면
빌드(`build-catalog` vite 플러그인)가 `dist/catalog/`에 파일 + `index.json`(목록)을
생성해 GitHub Pages에 함께 배포한다. **등록 = 파일 추가 + git push.**

- `id`: 파일명과 일치 (사용자 서가에 이 id로 설치됨 — 한 번 배포하면 불변)
- `version`: 개정 번호(기본 1). 내용을 고치면 올린다 → 설치자에게 "업데이트" 표시.
  **카드 id를 유지한 채** 고쳐야 설치자의 학습기록이 끊기지 않는다.
- 무결성은 `npm run lint`가 `src/data/`와 같은 규칙으로 검사한다.
- 주의: 카탈로그 문헌의 한자는 WenKai 서브셋(베이킹 콘텐츠 기준)에 없을 수 있어
  표지 제첨이 폴백 서체로 보일 수 있다 — 필요하면 `npm run font:subset` 재실행.
