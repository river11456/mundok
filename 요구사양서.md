# Project Name: Hanja-Crusher MVP (한문 강제 암기 로컬 웹 앱)

당신은 senior frontend engineer이자 한의학 학습 최적화 전문가입니다. 
제시된 사양서와 데이터 구조를 바탕으로, 아는 것은 빠르게 소거하고 모르는 것(구멍)은 실시간 대기열(Queue) 제어로 강제 반복 학습시키는 단일 파일(Single File) 형태의 로컬 웹 애플리케이션을 빌드해주세요.

---

## 1. 시스템 사양 및 아키텍처

### 1.1 기술 스택
- **Frontend:** HTML5, Tailwind CSS (CDN), Vanilla ES6+ JavaScript
- **Backend & Storage:** 백엔드 없음. 초기 데이터를 내장하고, 오답 횟수 및 진척도는 브라우저의 `localStorage`를 활용해 영속화.

### 1.2 핵심 UI/UX 규칙 (Keyboard-Driven)
- 마우스 조작을 완전 배제하고 오직 키보드로만 전이되는 상태 기반 뷰(State-based View)를 구현합니다.
- **Front View (질문 상태):** 화면 중앙에 순수 한자 원문(`front`)만 거대하게 표시. 하단에 `[Space] 정답 보기` 가이드 노출.
- **Back View (정답 상태):** `front`를 유지한 채, 하단에 한글 해석(`back`) 출력. 하단에 `[1] 어려움 | [2] 보통 | [3] 쉬움` 컨트롤 가이드 노출.
- **Key Mapping:** 
  - `Space Bar`: Front View -> Back View 전환 (Flip)
  - `1` 키: '어려움' 처리 후 다음 카드의 Front View로 전환
  - `2` 키: '보통' 처리 후 다음 카드의 Front View로 전환
  - `3` 키: '쉬움' 처리 후 다음 카드의 Front View로 전환

### 1.3 3단계 단순화 대기열 알고리즘 (Queue Logic)
현재 세션의 대기열인 `studyQueue` 배열을 유저의 피드백에 따라 실시간으로 변형합니다.
1. 카드가 출현하면 `studyQueue.shift()`로 먼저 큐에서 꺼냅니다.
2. 유저의 키 입력에 따라 아래와 같이 처리합니다.
   - **어려움 (1):** 카드의 `fail_count`를 1 증가시킴. 현재 남은 큐의 다음 위치 직후에서 최대 3번째 인덱스 이내(`[1] ~ [3]`)에 무작위(Random)로 강제 재삽입 (`splice` 연산).
   - **보통 (2):** 현재 남은 `studyQueue` 배열 총 길이의 정중앙 인덱스(`Math.floor(length / 2)`) 위치에 재삽입.
   - **쉬움 (3):** `studyQueue`에 다시 넣지 않고 완전히 제거(소거).
3. **세션 종료:** `studyQueue.length === 0`이 되면 학습이 종료되며, 오답률(`fail_count`가 높은 카드 순위)과 학습 완료 메커니즘을 보여주는 결과 리포트 UI를 출력합니다.
