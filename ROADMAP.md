## 배경 — 1학기 회고 (인터뷰 요약)

- 수업 형태: 교수님이 PPT에 한자만 나열해 띄우고 설명 → 학생들은 화면 바뀔 때마다 찍어서 아이패드로 옮겨 전자필기.
- 써머리 제도: 과목별 써머리 담당자가 수업 며칠 뒤 한문+해설+수업 내용을 정리한 PDF 배포.
- 1학기: 써머리 PDF를 LLM 도구로 수동 변환해 데이터화(당시 CSV). 초기 빌드와 학습이 병행돼 실학습 활용은 낮았음.
- **고충 1**: 찍고 → 옮기고 → 필기하는 실시간 워크플로가 너무 귀찮다. 단, 손으로 쓰는 공부는 무조건 필요하다는 전제.
- **고충 2**: 써머리는 좋은 원천이지만 며칠 늦고, 그것에만 의존할 수는 없다.

## 확정 결정

| 결정 | 내용 |
|------|------|
| 문헌 모델 유지 | 2학기도 한문 고전 강독 위주 → 기존 `DocJSON` 구조 유지. 수업 메타(날짜·주차)만 옵션 필드로 추가 |
| 2원천 인제스트 | **사진**(수업 당일·초안) + **써머리 PDF**(수일 후·보강 병합)가 같은 카드로 수렴. 카드 id 유지로 학습기록 안 끊김 |
| 손공부 재배치 | 수업 중 받아적기 → 복습 때 **회상하며 쓰기**(쓰기 학습 모드)로 이동. 수업 중엔 사진만 찍는다 |
| 디자인 방향 | **"클래식은 콘텐츠에만, 크롬은 모던하게"** — 한자 텍스트는 세리프로 정체성 유지, UI는 Mac/Apple 스타일 모던 미니멀 |
| 기기 전략 | PC 데스크톱 우선 + 아이패드 터치/펜슬 대응. 쓰기 모드가 아이패드 사용 동기가 되는 구조 |
