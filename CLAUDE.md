# CLAUDE.md — 文讀 저장소 규칙

프로젝트 개요·현황은 `PROGRESS.md`, 개선 목록은 `IMPROVEMENTS.md` 참고.

## 버전 규칙 (SemVer)

`package.json`의 `version`이 단일 출처다 — 홈 화면 하단에 자동 표시된다(`render.ts`가 import).

**코드·데이터가 바뀌는 모든 커밋은 같은 커밋 안에서 버전을 올린다.**

| 올릴 자리 | 기준 | 예 |
|-----------|------|----|
| MAJOR (x.0.0) | 저장 데이터·백업 포맷 비호환 변경 (localStorage 키·백업 JSON 구조 등) | 백업 포맷 v2→v3 |
| MINOR (1.x.0) | 사용자가 체감하는 기능 추가·개선 (feat) | 새 학습 모드, order 필드 |
| PATCH (1.0.x) | 버그 수정·리팩터·성능·빌드·의존성 (fix/refactor/chore) | 인덱스 통일, import 정리 |

- 예외: **문서만** 바꾸는 커밋(docs)은 버전을 올리지 않는다.
- 예외: 사용자 체감상 세대가 바뀌는 **대규모 리뉴얼 릴리스**는 비호환 변경이 없어도 MAJOR를 올릴 수 있다 — 로드맵 문서에 사전 명시된 경우만 (예: 2학기 리뉴얼 완료 시 `2.0.0`, 2026-07-18 확정).
- 콘텐츠(문헌 JSON) 추가·수정도 PATCH로 올린다 (새 문헌 통째 추가는 MINOR).
- 한 커밋에 여러 성격이 섞이면 가장 높은 자리 기준.
